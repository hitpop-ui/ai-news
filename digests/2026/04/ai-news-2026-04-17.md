### Claude Opus 4.7 正式リリース (Anthropic, 2026-04-16)
Anthropic の最新フラッグシップモデルが登場。コーディング性能が前世代比13%向上（93タスクベンチマーク）。
- SWE-bench Verified: 87.6%（前: 80.8%）、CursorBench: 70%（前: 58%）、GPQA Diamond: 94.2%
- **初の高解像度ビジョン**対応: 最大 2,576px (3.75MP = 前世代比約3.3倍)
- `xhigh` effort level 追加（high〜max の中間。速度と知性のトレードオフを細かく制御）
- Task Budgets（トークン消費のガイド）パブリックベータ開始
- 価格据え置き: $5/百万トークン (input) / $25/百万トークン (output)
- Amazon Bedrock / Google Vertex AI / Microsoft Foundry / GitHub Marketplace で同日 GA
- URL: https://www.anthropic.com/news/claude-opus-4-7

### Claude Code 2.1.111 / 2.1.112 (2026-04-16)
Opus 4.7 リリースに合わせた大型アップデート。ユーザビリティと自動化に関する多数の改善。
- `/ultrareview` 追加: 並列マルチエージェントによる包括的コードレビューを一発実行
- **Auto mode が Max 加入者に開放**（Opus 4.7 対応）。`--enable-auto-mode` フラグが不要に
- `/effort` を引数なしで呼ぶとインタラクティブスライダーが表示されるよう変更
- Read-only bash コマンド（glob パターン・`cd <project-dir> &&` 始まり）が permission prompt 不要に
- `/less-permission-prompts` スキル追加（transcript スキャンにより allowlist を自動生成）
- Plan ファイルがプロンプト内容に基づいて命名される（例: `fix-auth-race-snug-otter.md`）
- Windows: PowerShell tool の段階的ロールアウト開始（`CLAUDE_CODE_USE_POWERSHELL_TOOL` 環境変数）
- "Auto (match terminal)" テーマオプション追加
- `Ctrl+U` で入力バッファ全消去、`Ctrl+Y` で復元；`Ctrl+L` で全画面再描画
- 2.1.112: Opus 4.7 auto mode で "temporarily unavailable" エラーが出る問題を修正
- URL: https://code.claude.com/docs/en/changelog
